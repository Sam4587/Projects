## ⚠️ 注意事项

### 开发注意事项
1. **不要直接修改原始代码文件**
   - 先复制备份,然后修改
   - 便于回退和对比

2. **定期提交代码**
   - 每完成一个功能就提交
   - 避免大量代码丢失

3. **及时测试**
   - 修改后立即测试
   - 发现问题及时修复

4. **保留记录**
   - 记录每次修改的内容
   - 记录测试结果

### Git 注意事项
1. **第一次推送需要配置**
   - 设置 upstream 分支
   - 配置认证方式

2. **使用 Personal Access Token**
   - GitHub 已停止密码登录
   - Token 需要有 repo 权限

3. **提交信息规范**
   - 使用 `feat:` 前缀表示新功能
   - 使用 `fix:` 前缀表示修复
   - 使用 `docs:` 前缀表示文档
